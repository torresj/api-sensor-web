DOMÓTICA

La domótica y la automatización en el hogar es ya hoy en día una realidad. Las posibilidades solo están limitadas por nuestra propia imaginación. Desde los sensores mas simples hasta los sistemas mas complejos, todo puede ser conectado y controlado para hacer nuestras viviendas mas inteligentes gracias al IoT.

Easydom es una plataforma que nace con la inquietud de poner en practica diferentes tecnologías tanto software como hardware para construir un sistema que realice el proceso completo desde la lectura de un sensor hasta el almacenamiento y procesado. Además de esto, también permitirá interactuar con sistemas conectados a la red eléctrica.

Sensorización con lecturas de temperatura, humedad o calidad del aire entre otras.

Control de sistemas como la iluminación, la calefacción o cualquier otro que funcione mediante corriente electrica.

Presentación de datos mediante tablas y graficos para el seguimiento y análisis.

PRODUCTO LIBRE

Easydom es una plataforma basada en software y hardware libre y cuya única meta es la de practicar y probar nuevas tecnologías. Cualquiera que lo desee puede descargar, consultar y copiar el código de las distintas partes que conforman el proyecto.

El frontend de la plataforma está realizado con Angular 8 usando Material para el diseño. Es un proyecto sencillo en el que se intenta construir una interfaz amigable que permita acceder a una zona privada donde poder ver los datos de los sensores y actuadores asociados al usuario.

El backend de la plataforma se basa en una aplicación Java con Spring Boot, Spring cloud, y una base de datos MySql. El servidor debe cumplir con varios requisitos para permitir centralizar todas las operaciones de la plataforma:

  • Exponer una API Rest que permita la gestion de los diferentes modelos
  • Recibir mensajes MQTT enviados desde las estaciones de sensores
  • Enviar mensajes a las estaciones para utilizar los actuadores
  • Crear un web socket para realizar envío de información en tiempo real al frontend

El hardware de la plataforma se basa la utilización de módulos ESP32 con un firmware escrito en C (como cualquier proyecto con Arduino) para crear una base común a todas las estaciones, que permite conectarse a una red wifi para tener internet, y después realizar la conexión con el backend para mandar su información y la de los sensores que tengan conectadas. Practicamente se puede conectar cualquier sensor por lo que según las necesidades se conectan a unos u otros.

El módulo ESP32 se encarga de enviar mensajes en formato MQTT con las lecturas de sus sensores. También esperará recibir comandos en caso de que sea una estación actuadora.

La plataforma no requiere de ningún requerimiento hardware. Simplemente hay que enviar mensajes MQTT a una cola RabbitMQ, por lo que cómo leamos los sensores es irrelevante.

La infraestructura de la plataforma está formada por una serie de servidores que componen todo el sistema:

  • Sensor Server: Se trata del servidor que centraliza la recepción de datos y expone la API rest consumida por el frontend
  • Web Server: Es la parte frontend del proyecta, encargada de la aplicación web
  • Config Server: Para poder centralizar las configuraciones y separarlas del servidor principal, tenemos un Spring Cloud server
  • MySQL Server: Servidor de bases de datos en el que se almacenan los datos
  • RabbitMQ Server: Servidor de colas al que se envían los mensajes MQTT

Para poder desplegar todo facilmente, por un lado se ha creado un contendor para cada servidor creándolo con Docker y componiéndolo después todo con Docker Compose, y por otro lado se ha configurado Ansible, el cuál nos ayuda a realizar todo el provisionamiento y ejecución de Docker Compose en el entorno de producción.
